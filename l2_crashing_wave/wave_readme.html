Layer 2 Crashing Wave - LevelASM - Readme

by W4mp3/ShUriK KiD - Version 1.0

Preface

Don't be intimidated by this readme, it's long because everything this thing does is explained here. For general use just read Usage (using the template).

This LevelASM thing will create a crashing wave coming from the background and will wash all enemies (with configurable exceptions) and the player away if they are out in the open. I've included everything you need: graphics, palettes, blocks, Map16-data and a sample level.

Usage (using the template)

  1. Put üìÑ l2_crashing_wave.asm in the üìÅ level folder of UberASMTool and insert it.
  2. Put üìÑ block_l2_wave_safe.asm and üìÑ block_l2_wave_unsafe.asm into GPS üìÅ blocks folder and insert them.
  3. Import the ‚¨ú ExGFXAAA.bin into your ROM with LM. You can rename it but you need to change the ExGFX number inside the template level later.
  4. Import the Map16 data ‚¨ú map16.map16 to page 10. Anywhere else will work too, you just have to manually replace them in the template level.
  5. Import the example level üêâ example_level.mwl to your rom.
  6. Optional: Import the palette ‚¨ú palette.pal. You don't have to do this if you use the template level.
  7. Test the level, it should work now.
NOTE: By default you cannot use Item Memory Index 2 in your level, it uses that as free RAM by default.

Configuration

I tried to make as much as possible configurable. Let's just go through every define and what they do.

General Settings

Define Default value Description
!RAM_FreeRAM $1AF8|!addr Which Free RAM address this uses. By default it uses space used by Item Memory Index 2. Must be 32 bytes of free RAM.
!Setting_StrictMode !on Using strict mode will assume, that warnings are errors. This means that if the Patch is suspicious about your chosen values it will stop. Setting this to zero will continue the assembly (it will still display the warnings unless you set !DontIgnoreWarnings to 0). !StrictMode will be ignored when !DontIgnoreWarnings is set to 0.
!Setting_PerformanceMode !off By default this is disabled due to introducing some quirks and it will look worse. Set this to 1 to run only every second frame. The timers will be automatically adjusted for you so if you put in $003D (61 Frames) it will halve the time to $001E (30 Frames, value will be always rounded down). Known Quirks: The wave crashes downwards a bit faster, very fast Sprites might be ignored.
!Setting_LavaMode !off Sets the Wave to Lava mode. Everything the wave touches will be destroyed instead of being flushed downwards. The Sprite will not respawn as it is then marked as killed and not despawned. The player will die instantly when touching the wave (!Settings_WaveInstantKillPlayerWhenTouched = !on) and you are not in a safe area (Yoshi won't save you, too). Note: You might want to modifiy which sprites are immune to the wave.
!Setting_EnableInteraction_Player !on Allows the Wave to interact with the player. You can disable this by setting this to 0.
!Setting_EnableInteraction_Sprites !on Allows the Wave to interact with sprites. You can disable this by setting this to 0.
!Setting_RidingYoshiDoesntAffectPlayer !on When riding Yoshi the wave cannot harm the player (Yoshi + Mario are heavy). Disable this if you do not want Yoshi to change the behaviour. You should remove Yoshi from the immune_sprites list, too, or prepare for Yoshi to drop through the floor. If !Setting_LavaMode is on this is forced to !off.
!Setting_CustomSpriteHandling !off Disables Custom Sprite handling meaning it uses the settings from regular Sprites. Example: Sprite 0E (Keyhole) is by default not affected by the wave. If you set this to 1 it'll make Custom Sprite 0E immune, too.
!Setting_WaveInstantKillPlayerWhenTouched !off Player will be instantly killed when touching the wave. !Setting_LavaMode is setting this to !on, if it is active. If you set !Setting_RidingYoshiDoesntAffectPlayer to !on this will not kill the player!
!Setting_WaveInstantKillSpriteWhenTouched !off Sprites will be instantly killed when touching the wave. !Setting_LavaMode is setting this to !on, if it is active. I recommend leaving this to !on if you want to be fair to the player, otherwise sprites will still interact with (and possibly hurt) the player.
!Setting_EmulateTide !on Emulates the Tide on Layer 3. Normally you cannot use Tides in a Layer 2 Level, this is a work around that enables you to do that.
!Setting_EmulateTide_Height $C1 Where the Tide is located on screen. By default this is only the bottom most row.
!Setting_DangerousWaterCurrent !off Makes swimming inside water very dangerous by pulling the player downwards very quickly. You still can escape it, though.
!Setting_AllowSFX !on Setting to globally enable sound effects. Setting this to !off will disable all sound effects
!Setting_AllowSFX_Warning !on Enable warning sound effect when Wave is approaching. Which sound is played is set by !Wave_Phase_01_Warning_SFX.
!Setting_AllowSFX_Wave !off Enable wave sound effect when wave is on screen. For each Phase you can set a different sound effect. !Wave_Phase_02_Wave_SFX.

Wave Phase settings

The Wave is split into multiple phases:

  • Phase 00: Idle. As the name suggests nothing happens here. This is the time between the wave being gone and the next phase.
  • Phase 01: Warning. Shows a warning/plays the warning sound effect. Anything else is the same as phase 00. This helps the player to know the wave is going to come very soon.
  • Phase 02: Rise. The wave rises up to !WAVE_MAX_Y.
  • Phase 03: Move sideways. The wave moves sideways up to !WAVE_MAX_X.
  • Phase 04: Fall. The wave falls downwards taking sprites and the player with it.
  • Phase 05: Reset. The wave moves back to it's starting position.
Define Default value Description
!WAVE_MAX_Y $017F How far up the wave goes in pixels before it moves sideways. Don't set this to less than 224 ($00D0). While values above $0200 will work do not go higher than your level height in pixels or else you will get garbage tiles.
!WAVE_MAX_X $0100 How far the wave goes sideways. While you can set this to 0 this will look weird. Do not set this to more than the width of the level in pixels (basically number of screens * $100 - $100). Maximum value is $0F00, which is the maximum any Layer 2 Level can scroll horizontally.
!WAVE_HITLINE_Y $0110 Where the Wave is considered to hit something. This should be lower than !WAVE_MAX_Y.
!Wave_Phase_00_Init_Time $00B4 How many Frames the wave waits before the wave appears for the first time (= after level load).
!Wave_Phase_00_Time $00B4 How many Frames the wave waits before the wave appears for each cycle (except for the first time).
!Wave_Phase_01_Time $0061 How many Frames the warning should be shown/heard.
!Wave_Phase_01_Warning_SFX $01 The sound effect number to play repeatedly. Don't forget to set !Wave_Phase_01_Warning_SFX_Bank! By default this plays the coin sound effect.
!Wave_Phase_01_Warning_SFX_Bank $1DFC Which sound bank to use.
!Wave_Phase_01_Warning_SFX_Speed $20 How many frames between each warning sound effect. Set this to 0 if you just want to play the sound effect once.
!Wave_Phase_02_Wave_SFX $00 The sound effect the wave makes. By default there is no SFX set.
!Wave_Phase_02_Wave_SFX_Bank $1DFC Which sound bank the sound effect of the wave uses.
!Wave_Phase_02_Wave_SFX_Speed $04 Play the wave sound effect every given frame amount.
!Wave_Phase_02_Wave_Init_Speed $0006 How fast the Wave rises initially. Values above $0010 can produce garbage tiles.
!Wave_Phase_03_Wave_SFX $00 The sound effect the wave makes. By default there is no SFX set.
!Wave_Phase_03_Wave_SFX_Bank $1DFC Which sound bank the sound effect of the wave uses.
!Wave_Phase_03_Wave_SFX_Speed $04 Play the wave sound effect every given frame amount.
!Wave_Phase_04_Wave_SFX $00 The sound effect the wave makes. By default there is no SFX set.
!Wave_Phase_04_Wave_SFX_Bank $1DFC Which sound bank the sound effect of the wave uses.
!Wave_Phase_04_Wave_SFX_Speed $04 Play the wave sound effect every given frame amount.

Etc. Settings

Here are settings to disable some features this patch has and couldn't find a fitting spot in the other define tables. Normally you do not need to modify anything here.

Define Default value Description
!DontIgnoreErrors !on Do not ignore errors caused by the user (you). DO NOT SET THIS TO 0 TO DISABLE ERROR CHECKS.
!DontIgnoreWarnings !on Do not ignore Warnings. DO NOT SET THIS TO 0 TO DISABLE WARNING MESSAGES.
!DEBUG !off Some Debug features displaying !FreeRAM on screen (Layer 3). Useful if you want to modify the wave and want to see some values. It doesn't update every frame for reasons.

Sprite Settings

There are two tables for Sprites, one for regular and one for custom sprites. Both tables decide if a sprite is affected by the wave. If you set it to $00 the sprite will be affected, $01 will make the sprite immune to the wave.

Usage (manual setup for custom wave)

NOTE: Avoid manual setup if you can, it's a bit complicated but allows you to do more with the wave.

I will go through the whole setup process I used for the template. The Level code will throw errors if you have entered something weird, don't worry just read the error message and you should be good to go.

  1. Set the level to Layer 2 (Level Modes 01, 02, 0F, 1F should work, the template uses 0F). While were at it: Set the Level height to the correct value, getting that is simple: The wave must be at least two screens high ($1C (28) Tiles) which would be Horizontal Level Mode 01 plus how big the head of the wave is (see image in next step). The head of the wave is the part which should hide everything behind it. In my case this would be 9 additional tiles or Horizontal Level Mode 05.
  2. Next we build the Wave on Layer 2. I just show you the result for my template (I included some lines to illustrate better):
    Set the wave head tiles with priority to make it go over sprites and players.
  3. Check if Layer 2 Scrolling is disabled (if it isn't, disable it):
  4. Now add the Code to the Level via UberASMTool and test the level. It should work now but where the Wave hits you is not in the correct spot, the wave might go too high producing garbage tiles on the bottom of layer 2 and/or the wave doesn't go high enough.
  5. Insert the two provided blocks üìÑ block_l2_wave_safe.asm and üìÑ block_l2_wave_unsafe.asm. Put the safe block on the starting position of the player and test the level. You shouldn't be affected by the wave now if you test the level.
  6. Now we need to modify the code with the correct values to make the wave go the correct height and the correct height where the wave pulls you. I recommend putting two bigger sprites in the level in a box so you can see if they are getting pulled down at the correct time. In üìÑ l2_crashing_wave.asm we go to ;; Advanced Settings. Below that should be ;; Wave, there we have three defines:
    • !WAVE_MAX_Y is how far the Wave needs to scroll vertically. You can get this value very easily by making a selection from the top to the bottom of the wave head and multiply it by 16. In my example this is 23 Blocks, which translates to 368 pixels ($0170). I recommend to subtract 1 from this, SMW puts by default Layer 2 1 pixel higher.
    • !WAVE_MAX_X is how far the wave scrolls horizontally. In my case this is very simple, our wave is 2 screens wide. Just measure from the left-most block to the wave head and multiply that by 16, this will always give you the correct value. In my example this is 256 pixels ($0100).
    • !WAVE_HITLINE_Y is where the wave is "sticky". This should be smaller than !WAVE_MAX_Y. You can get the value easily by measuring it the same way you measured the !WAVE_MAX_Y, except you just go a bit higher. In the template this is 17 blocks from the top so I put $0110 there.
  7. Done! Test it and it should be correct now.

Troubleshooting

My game slows down all the time!

Try setting !Setting_EnableInteraction_ClusterSprites and !Setting_EnableInteraction_ExtendedSprites to 0. If you still have slow down try setting !Setting_PerformanceMode to 1.

My custom sprite is affected by the wave, it shouldn't be!

Add the sprite number your custom sprite uses to the table immune_custom_sprites (also set !Setting_CustomSpriteHandling to !off !). Extra bit is not supported nor extension bytes due to performance and insertion size.
